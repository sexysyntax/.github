# SexySyntax Discussion Guide

Welcome to SexySyntax Discussions. This is the community space for the SHNWAZ DEV developer ecosystem: public APIs, automation tools, website work, docs, security notes, release planning, and project ideas.

The goal is simple: keep every post clear, useful, respectful, and easy for maintainers to act on.

## Start Here

- Read the category guide before posting.
- Search existing discussions before opening a new one.
- Use a short, specific title.
- Add enough context for another person to understand the request without guessing.
- Keep one topic per discussion.
- Share links, screenshots, logs, or reproduction steps when they help.
- Do not post secrets, tokens, passwords, private keys, cookies, or personal data.

## Categories

| Category | Use it for | Good title example |
| --- | --- | --- |
| Announcements | Maintainer updates, launches, changes, and release notes. | `Release: syntax homepage refresh` |
| General | Open chat, planning, coordination, and project questions. | `Planning the next API docs update` |
| Ideas | Feature requests, roadmap suggestions, automation concepts, and API improvements. | `Idea: add a status endpoint example` |
| Polls | Quick votes when the team needs a simple decision. | `Poll: which docs section should ship first?` |
| Q&A | Help requests that need a clear answer. | `How do I run the site locally?` |
| Show and tell | Demos, builds, launches, experiments, and community wins. | `Show and tell: new static site preview` |

## Posting Rules

1. Be respectful and professional.
2. Stay on topic for the category you choose.
3. Do not spam, advertise unrelated services, or duplicate the same post.
4. Do not share private credentials or sensitive information.
5. For bugs, include what happened, what you expected, and steps to reproduce.
6. For ideas, explain the user problem and the value of the suggestion.
7. For Q&A, mark the answer when your question is solved.
8. For announcements, keep the message factual and include links to the relevant repo, release, or pull request.
9. For polls, explain the decision, the options, and when voting should close.
10. Maintainers may edit titles, move categories, close duplicates, or remove unsafe content.

## Maintainer Response Flow

- Triage new posts regularly.
- Move posts to the right category when needed.
- Ask for missing details instead of guessing.
- Link issues or pull requests when a discussion becomes actionable work.
- Close stale or duplicate discussions with a short explanation.
- Keep announcement posts short, dated, and linked to the source of truth.

## Welcome Message

Welcome to SexySyntax. If you are here to build, learn, ask, suggest, or share something useful, you are in the right place.

Choose the best category, write clearly, and help the team understand what should happen next. Good discussions make the organization easier to maintain and help new members join the work faster.
