# Contributing to SexySyntax

Thank you for helping improve SexySyntax.

## How to Contribute

1. Open a discussion or issue before starting a large change.
2. Fork the repository and create a focused branch.
3. Keep changes small, readable, and connected to one purpose.
4. Update docs when behavior, setup, routes, commands, or workflows change.
5. Run available checks before opening a pull request.
6. Explain what changed, why it changed, and how it was tested.

## Pull Request Rules

- Use a clear title.
- Keep unrelated refactors out of the pull request.
- Include screenshots for UI changes.
- Include reproduction steps for bug fixes.
- Do not commit secrets, generated credentials, or local environment files.
- Be ready to respond to review comments.

## Maintainer Checklist

- README is current.
- Security policy is current.
- Actions use least-privilege permissions.
- Discussions and issues are routed to the right category.
- Releases and announcements link to the source repository.
